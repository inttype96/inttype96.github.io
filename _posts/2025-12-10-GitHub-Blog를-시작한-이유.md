## GitHub Blog를 시작하게 된 이유

안녕하세요! 개발자로서의 성장 과정을 기록하고 공유하기 위해 **GitHub Blog**를 시작하게 된 **inttype**입니다.

사실 궁금해하실 분은 많지 않겠지만, 앞으로 어떤 글을 쓰게 될지 소개하는 겸 나중에 초심을 돌이켜보기 위해 왜 GitHub Blog를 선택했는지 가볍게 정리해보려고 합니다.

저는 이전에 티스토리에서도 블로그를 운영했지만, 기술적인 글보다는 알고리즘 풀이 위주로 글을 작성했었습니다. 
그러다 보니 *“하루에 한 문제씩 풀고 글을 올려야 한다”* 는 부담감이 생겼고, 시간이 부족한 날에는 쉬운 문제만 풀고 형식적으로 풀이를 적는 등 기록 자체가 의미를 잃었다고 느꼈습니다.

이런 이유로 점점 의욕이 줄어들었고, 바쁜 현생과 함께 블로그는 자연스럽게 중단되었습니다.

그래서 이번에는 알고리즘 풀이보다는 **실제 개발 경험, 기술적인 내용, IT 트렌드, 그리고 개인적인 잡담까지** 자유롭게 기록할 수 있는 블로그를 만들고자 새로 GitHub Pages를 사용해 블로그를 시작하게 되었습니다.

---

## 그런데 왜 GitHub Pages인가요?

블로그를 다시 시작하려고 여러 플랫폼을 비교해보았고, 네이버, 티스토리, 벨로그, GitHub Pages 중 GitHub Pages를 선택하게 되었습니다.

## 플랫폼 비교

| 플랫폼 | 장점 | 단점 |
|-------|------|------|
| **네이버 블로그** | - 검색 노출(SEO)이 국내 사용자에게 유리<br>- 앱/웹 편집기가 편리 | - 기술 블로그 분위기와는 다소 거리가 있음 |
| **티스토리** | - 커스터마이징 자유도 높음<br>- 구글 검색 노출 강함<br>- 광고 수익화 가능 | - 운영 안정성 이슈 지속 (카카오 분사 등) |
| **벨로그 (Velog)** | - 개발자 친화적인 글쓰기 환경<br>- 마크다운 기반 | - 디자인/구조 커스터마이징 제한<br>- 광고/운영 정책의 변동 가능성 |
| **GitHub Pages** | - Git 기반 버전 관리<br>- 원하는 테마·구조로 직접 제작 가능 (Jekyll/Hugo) | - 설정 난이도 있음<br>- 비공개 기능 사실상 없음 |

<details>
<summary>개인적인 선택 이유</summary>

사실 위 비교 내용은 검색만 해도 쉽게 찾을 수 있는 정보이고, 개발 블로그를 시작하려는 분들이라면 대부분 알고 있는 내용일 것입니다.
  
개인적인 생각으로는 네이버 블로그는 후원·협찬 관련 게시글이 많아지면서 예전보다 정보 신뢰도가 떨어지는 느낌을 받았습니다. (개발 블로그와는 직접적인 연관은 없지만, 전체적인 분위기가 그렇게 느껴졌습니다)

티스토리는 예전에 사용해봤기 때문에 새로 시작한다는 느낌이 덜했고, 운영 이슈가 계속 언급되는 점도 고려해 선택지에서 제외했습니다.

벨로그는 플랫폼에서 광고를 붙인다는 점이 개인적으로는 조금 거부감이 들었습니다. 글을 읽을 때 광고가 방해요소라고 생각하는 편이라, 작성자가 수익을 위해 직접 광고를 넣는 건 이해하지만 플랫폼 차원에서 제공하는 광고는 조금 아쉬웠습니다. 

GitHub Pages는 Git 기반으로 버전 관리를 할 수 있다는 점이 마음에 들었고, 아직 마크다운 작성이 완전히 익숙하지는 않지만 블로그를 운영하면서 자연스럽게 익숙해질 수 있을 것 같다는 점도 긍정적으로 느껴졌습니다.

이런 점들을 종합적으로 고려해보니, GitHub Pages가 가장 제 취향과 목적에 잘 맞다고 생각해 선택하게 됐습니다.

## 앞으로의 계획

이 블로그에는 주로 다음과 같은 내용들을 담을 예정입니다.

- **개발 공부 기록** : 새롭게 알게 된 기술이나 개념 정리  
- **트러블슈팅** : 개발 중 마주친 문제와 해결 과정  
- **회고** : 프로젝트 경험이나 월간·연간 회고  
- **일상** : 개발 외의 소소한 이야기들  

최소 격주에 한 번은 꼭 작성해보려고 합니다.
